# Agent 命令用户体验问题分析

## 一、核心问题：上下文恢复功能

### 问题1：ID显示与恢复不一致

| 操作 | 显示内容 | 需要输入 |
|------|----------|----------|
| `AGENT 上下文 列表` | `20260216` | - |
| `AGENT 上下文 恢复 <ID>` | - | `abc123-def456_20260216_143022` |

**影响**：用户无法根据列表输出输入正确的恢复ID

### 问题2：MC聊天无法复制粘贴

- 完整ID长度约20+字符
- 需手动逐字符输入
- 极易输错

### 问题3：同日期多会话无法区分

```
显示: [20260216] 玩家A | 10条消息
显示: [20260216] 玩家B | 5条消息
```

同一天多个会话的ID显示完全相同，无法区分

### 问题4：删除功能同样存在

```
AGENT 上下文 删除 <ID>
```

---

## 二、其他问题

### 命令语法提示不足

```
当前: AGENT 上下文 恢复 <ID>
问题: 未说明ID从哪里获取、格式是什么
```

### 错误提示不够友好

```
当前: "会话不存在: xxx"
问题: 用户不知道如何查看可用会话
```

---

## 三、优化建议

### 建议1：短ID显示

```
原: 20260216
改: 20260216_a3b7  (日期 + 连接ID后4位)
```

### 建议2：支持序号操作

```
AGENT 上下文 恢复 1
AGENT 上下文 删除 2
```

### 建议3：增强帮助提示

```
恢复 <ID或序号> - 从列表选择或输入ID
删除 <ID或序号> - 从列表选择或输入ID
```
