# 05 — 支持玩家相对坐标与审批前落点锁定

**What to build:** 让玩家可以使用“前/右/上”偏移查询和放置方块，同时确保多人会话隔离且玩家在审批期间移动不会改变已授权落点。

**Blocked by:** 04 — 完善单格条件写入、状态与兜底保护.

**Status:** ready-for-agent

- [ ] `inspect_block` 和 `edit_blocks(mode=place)` 支持 `player_relative` 坐标模式及 `forward/right/up` 偏移。
- [ ] 原点为当前事件玩家脚部方块，水平朝向吸附到东南西北，结果返回捕获的锺点和最终绝对坐标。
- [ ] 相对操作只绑定当前事件的 `player_name` 和当前维度，不接受目标玩家或选择器。
- [ ] 相对坐标在审批前的只读预检中解析为绝对坐标，审批摘要只显示最终落点与必要锺点信息。
- [ ] 规范化调用按 `(run_id, tool_call_id, 原始参数哈希)` 缓存并跟随现有 120 秒审批 TTL。
- [ ] 审批恢复使用原已锁定坐标，不因玩家后续移动、转向或切换会话而重新解析。
- [ ] 审批过期或断线会清理缓存，后续操作必须重新发起。
- [ ] 多玩家并发测试证明坐标、审批缓存和工具审计均使用当前事件的 `player_name`。
