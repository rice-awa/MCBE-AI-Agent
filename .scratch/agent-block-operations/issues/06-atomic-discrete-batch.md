# 06 — 支持原子离散批量放置

**What to build:** 让 MCBE Chat Agent 能够在多个绝对或玩家相对位置放置同一 permutation，通过整批预检、写前复核和失败回滚避免留下半成品。

**Blocked by:** 05 — 支持玩家相对坐标与审批前落点锁定.

**Status:** ready-for-agent

- [ ] `edit_blocks(mode=batch)` 接受整批共享的坐标模式、type ID、states、覆写策略和可选 `expected_previous`。
- [ ] positions 列表只包含位置，整批放置同一 permutation；不允许混合坐标模式或方块类型。
- [ ] 默认上限为 256 个位置、配置硬上限为 1024，超限在写入前返回 `LIMIT_EXCEEDED`。
- [ ] 审批前对所有位置完成参数、区块、前置条件和数据组件保护预检，任一预检失败都不写入。
- [ ] 执行前复核所有目标，任一并发变更使整批返回 `PRECONDITION_CHANGED` 且不写入。
- [ ] 执行中任一位置失败时恢复本批已写入的普通方块，并逐项验证回滚结果。
- [ ] 成功结果返回数量、旧类型统计、验证摘要和有界样本，不因批量大小无限增长。
- [ ] 集成测试覆盖预检失败无变更、并发变更、中途失败回滚和幂等恢复不重复执行。
