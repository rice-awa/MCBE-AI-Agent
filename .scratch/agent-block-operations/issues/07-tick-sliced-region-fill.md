# 07 — 支持分 Tick 区域填充

**What to build:** 让 MCBE Chat Agent 能够安全填充一个长方体区域，在不长时间占用游戏主线程的前提下完成预检、保护已有方块并返回可理解的统计结果。

**Blocked by:** 05 — 支持玩家相对坐标与审批前落点锁定.

**Status:** ready-for-agent

- [ ] `edit_blocks(mode=fill)` 接受共享坐标模式下的 `from`/`to` 角点，并归一化为最小/最大绝对坐标。
- [ ] 默认体积上限为 4096 格、配置硬上限为 16384，超限在扫描前返回 `LIMIT_EXCEEDED`。
- [ ] 预检每 Tick 最多处理 128 格，且不阻塞 bridge router 处理其他事件。
- [ ] 默认仅填充空气并统计跳过的非空气；`expected_previous` 在 fill 中作为 type ID 和部分 states 的 include 过滤条件。
- [ ] 显式覆写的影响集合中发现任一数据组件方块时，整次填充拒绝且不写入。
- [ ] 执行前复核所有将受影响的位置，并发变更使整次操作返回 `PRECONDITION_CHANGED` 且不写入。
- [ ] 成功结果返回实际改变数量、跳过数量、旧类型统计和有界异常样本。
- [ ] 执行开始后的异常返回 `STATE_UNKNOWN`，不声称全量回滚、不标记可自动重试。
- [ ] 测试覆盖默认空气过滤、条件过滤、受保护方块、未加载区块、越界、扫描分 Tick 和未知外部状态。
